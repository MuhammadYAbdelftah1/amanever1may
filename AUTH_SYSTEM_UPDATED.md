## 📊 أنواع المستخدمين (5 أنواع):

### 1️⃣ **الزوار** (Visitors) ← جديد ✨
- **الأيقونة:** Users (مجموعة أشخاص)
- **الوصف:** دخول الزوار والمستخدمين الجدد
- **طرق الدخول:**
  - رقم الجوال + Password
  - البريد الإلكتروني + Password
  - اسم المستخدم + Password
  - أو دخول سريع بـ OTP

### 2️⃣ **المشتركين** (Members)
- **الأيقونة:** User
- **الوصف:** أصحاب بطاقات أمان إيفر
- **طرق الدخول:**
  - رقم البطاقة + Password
  - رقم الجوال + Password
  - البريد الإلكتروني + Password
  - اسم المستخدم + Password

### 3️⃣ **الأطباء** (Doctors)
- **الأيقونة:** Stethoscope
- **الوصف:** منصة الأطباء
- **طرق الدخول:**
  - رقم الهوية + Password
  - رقم الجوال + Password
  - البريد الإلكتروني + Password

### 4️⃣ **التجار** (Merchants)
- **الأيقونة:** Store
- **الوصف:** منصة التجار
- **طرق الدخول:**
  - رقم الهوية + Password
  - رقم الجوال + Password
  - البريد الإلكتروني + Password

### 5️⃣ **المسوقين بالعمولة** (Affiliates)
- **الأيقونة:** TrendingUp
- **الوصف:** منصة المسوق بالعمولة
- **طرق الدخول:**
  - رقم الهوية + Password
  - رقم الجوال + Password
  - البريد الإلكتروني + Password

## 🎯 الفرق بين الزوار والمشتركين:

### **الزوار:**
- مستخدمين جدد أو زوار الموقع
- ليس لديهم بطاقة أمان إيفر
- يمكنهم التسجيل وتصفح الخدمات
- الدخول بـ: Mobile / Email / Username + Password

### **المشتركين:**
- لديهم بطاقة أمان إيفر فعلية
- يمكنهم الدخول برقم البطاقة
- لديهم اشتراك نشط
- الدخول بـ: Card Number / Mobile / Email / Username + Password

## 🔐 للـ Backend Developer:

### API Login المحدث:

```typescript
POST /api/auth/login
Body: {
  identifier: string,  // Card/ID/Mobile/Email/Username
  password: string,
  userType: 'visitor' | 'member' | 'doctor' | 'merchant' | 'affiliate',
  rememberMe: boolean
}

Response: {
  success: boolean,
  token: string,
  user: {
    id: string,
    type: 'visitor' | 'member' | 'doctor' | 'merchant' | 'affiliate',
    name: string,
    // ... other user data
  }
}
```
